- Transpose의 기본 성질들 (Obvious Properties)
    
    ```yaml
    1. Distributive Property over Addition :
    
    	설명 :
    	두 같은 크기의 matrix 더할 때
    	더한것을 transpose하나, transpose한 것을 더하나 같다.
    	
    	왜 되나 :
    	
    	matrix addition은 같은 자리의 element끼리 더함.
    	dot product처럼 곱하고 더하는 것이 아니다.
    	따라서, transpose하여도 같은 element들이 더해진다.
    
    2. Scalar Multiplication and the Transpose :
    
    	설명 :
    	Scaling 하고 transpose하나, transpose한 것을 scaling하나 같다.
    	
    	왜 되나 :
    	
    	너무 당연하다.
    	Scaling한다는 것은 matrix 그 자체의 모든 요소들에 상수만큼 곱하는 것.
    	자리 바꿔서 곱하나, 곱한뒤 자리바꾸나
    	모든 요소들이 x배 된다는 상황은 같다.
    	
    	왜 되나 2 :
    	
    	transpose 자체의 규칙 때문인 것 같다.
    	transposing한다는 것은, 정보들이 표기되는 위치만 바꾼 것.
    	연산할 때 동일한 것들만 연산된다면 정보가 바뀔 이유가 없다.
    	
    3. Double Transpose :
    
    	 설명 :
    	 Transpose한 것을 다시 Transpose하여도 원래의 것이 된다.
    	 
    	 왜 되나 :
    	 너무 당연한 것 같다..
    	 행이 열 되었다가. 아무런 연산도 취하지 않고, 다시 열들을 행으로 되돌리니.
    	 좀 더 깊이 파보면 더 논리적인 규칙이 나오겠지만 괜찮을 듯.
    	 
    
    4. Distributive Property over Subtranction :
    
    	 설명 :
    	 같은 크기의 matrix A - B를 할 때,
    	 Transpose하고 빼도, 빼고 transpose해도 결과가 같다.
    	
    ```

    1. Matrix Product Transposition
    
    	: 곱해서 transpose한 것과, 각각 transpose 해서 곱한 최종 matrix C가 같다.
    	
    	이해 :
    	
    	실제로는 단순히 각각 transpose한 뒤 원래 순서대로 multiply하는 것과는 다름.
    	
    	Bt At로 뒤집어서 연산해야 하는 이유.
    	
    	우리는 행과 열을 transpose하였다.
    	행렬 곱하기는 행과 열을 각각 곱한다.
    	그대로 곱하면 위치가 변환된 요소들끼리 곱해질 수 있다. (A의 행과 B의 열)
    	이러면 원하는 요소들끼리 곱해지지 않는다.
    	
    	따라서 B의 행과 A의 열이 곱해질 수 있도록 연산을 AB -> Bt At로 바꾼다.
    	그러면 transposed된 상태의 최종 matrix를 얻을 수 있다.
