# 34 Diagonal Matrices (9분)

- diagonal matrices 정의
    
    ```yaml
    대각 행렬은 다음과 같은 조건을 만족하는 행렬 :
    
    행렬 A가 대각 행렬일 때, 모든 i ≠ j에 대해 aᵢⱼ = 0
    
    - 반드시 정사각 행렬(n×n)
    - 주대각선(main diagonal) 외의 모든 원소가 0
    
    ⎡3 0 0⎤
    ⎢0 7 0⎥ 
    ⎣0 0 4⎦
    
    A = 벡터의 집합 관점으로 본다면 :
    	각 위치에 corresponding하는 차원에 대한 정보만 가진 matrix.
    		row 1 = 1차원 정보만
    		row 2 = 2차원 정보만
    		row 3 = 3차원 정보만...
    	
    	원래 Matrix라는 것이, col number가 혹은 관점에 따라 row number가,
    	몇 차원인지에 대한 정보를 hold하고 있음.
    	
    	그래서 이걸 :
    	"linear equation 으로 풀면"
    	
    		각각 x y z 위치로 배치하지 않는가?
    		   [1 2 3]
    		   
    		   = 1x 2y 3z
    		   
    	그래서 equation 풀 때는 matrix를 row vector 관점으로 나열하고,
    	연산할때는 col vector 관점으로 나열해서 계산한다.
    ```

- diagonal matrices는 늘리기만 하고 회전하지 않음
    
    ```yaml
    왜 늘리기만 하냐고 :
    	
    	"해당 차원에 대한 정보를 input하였을 때, 해당 차원에 대한 정보만 뱉어내니까"
    
    각 input하는 정보에 대하여, corresponding하는 차원에 대한 정보만 변형하도록 하였다.
    따라서, 당연히 scalar배 되기만 한다.
    
    방향전환하려면 ? :
    
    	"각 축의 관계식이 바뀌어져야한다"
    	
    	무슨말이냐면, x가 하나 바뀔 때 y가 2개 바뀌던 비율관계가 변화되어져야함.
    	
    	그렇게 되려면 :
    		
    		x를 input하여 변환하였을 때 새로운 x,y값이 나오고
    		y를 input하여 변환하였을 때 새로운 x,y값이 나와서,
    		이들이 더해지면, 최종 vector의 x,y 관계식이 완전히 변화하게 된다.
    			그럼 방향전환이 일어남.
    
    ** 정리 필요
    	1에 대응되는 차원 1의 정보만 x배 변환하도록 하였다.
    	1에 대응되는 2차원, 3차원 정보는 0이다.
    	따라서 1차원에 대한 정보는 오로지 1-1에서만 다룬다.
    	기존의 각 차원의 정보들을 x배 하는 것은 가능.
    
    	    1차원 2차원       정보1  정보2
    	A = 3     5       B = 2    0
    			4     6           0    2
    			
    
    정보 1은 오로지 1차원에 대한 정보만 담고 있음.
    정보 1을, 즉2를 변환기 A에 넣으면,
    1차원 변환표에 따라 변환되어 나오는데.
    2차원 정보를 담고 있지 않으므로 정보가 추가적으로 합성되지 않음.
    1차원 정보만 2개 나옴. 6이랑 8.
    ```
