# 53 Positive Definite and Negative Definite Matrice

$$
f(x)= a x_1^2 + (b+c)x_1 x_2 + d x_2^2

$$

```yaml
"Matrix A의 성질에 따라 f(x)의 최종 부호가 결정될 수 있다."

matrix multiplication의 함수 (quadratic)

	- 일반 matrix도 변환 함수로 볼 수 있다.
	- 이번엔 matrix를 활용한 2차 함수 이다.
		
같은 행렬 A에 대해 이차형식 함수의 형태 :

	Function의 형태 : "f(x) = xT A x"
	
  = 2x₁² + 2x₁x₂ + 2x₂²

	x = n dimension vector (column vector)
	A = n x n matrix	

함수 계산 결과값에 따른 분류 :

A의 성질에 따라, f(x)는 언제나 positive, negative, both가 될 수 있다.

	1. Positive Definite :
			
			모든 x ≠ 0에 대해 x^T A x > 0

	2. Negative Definite :
			
			모든 x ≠ 0에 대해 x^T A x < 0

	3. Positive Semidefinite :
			
			모든 x ≠ 0에 대해 x^T A x >= 0
			0값도 허용하는 matrix
						
	4. Negative Semidefinite :
	
			모든 x ≠ 0에 대해 x^T A x <= 0
			0값도 허용하는 matrix
			
	5. Indefinite :
			
			input vector에 따라 positive, negative 여부가 달라짐.
			saddle point (기하학적으로 안장같이 생긴 그래프를 생성, 그것이 무엇인지는 생각이 오염될까봐 찾아보지 않았다)
			
```

```yaml
표기법 :

- Positive Definite: A > 0
- Positive Semidefinite: A ≥ 0
- Negative Definite: A < 0
- Negative Semidefinite: A ≤ 0
```

```yaml
Principal Minors :

	** 양의 정부호 행렬인지 판단하는 한 가지 방법은 모든 주대각식의 값이 양수인지 확인하는 것입니다.
	
```

```yaml
Mechanism of Function :

	지금 뭘 하고 있나요? :
		
		Quadratic Equation, 그것의 계산하는 순서와 그 계산결과의 의미를 탐구하고 있습니다.
		그러던 와중에, 내가 알고 있던 개념들에 혼동이 왔습니다.
		특히, matrix multiplication을 변환관점에서 보던 나의 방식이 틀렸는지 의심하게 됩니다.
		
		Mechanism을 탐구하던 와중, 발생한 의문은 다음과 같습니다.
		
	의문 :
	
		1) Quadratic Equation에서 xAx^T를 계산하는 이유가 무엇일까요?
			
		
		2) x와 A의 연산에서, x를 x^T로 변환하였을 때
			
			2-1) xTA로 계산을 바꾸어야 하나요?
			2-2) xT 와 x가 담고 있는 정보는 다른가요?
	
	
	의문에 대한 이해 :
	
		의문 2 : x와 A의 연산에서, x를 x^T로 변환하였을 때 
		
		2-1) xTA로 계산을 바꾸어야 하나요?
		
		법칙에 대한 암기가 명확하지 않아 혼동되었었습니다.
		원래 Transpose규칙은 Ax일 때, xT AT = (Ax)T 입니다.
		내가 이전에 배웠던 규칙에서는, x만을 transpose하여 연산하는 규칙 같은 것은 없었습니다.
		
		x를 transpose한다고 해서, 그것이 A의 앞으로 가야하고 이런 규칙은 없었습니다.
		
		따라서, quadratic equation에서 transposed 된 x를 A와 연산하는 것은,
		
		
		2-2) xT와 x가 담고 있는 정보는 다른가요?
	
		경우에 따라 다릅니다.
		일반적인 Transposition은 정보를 바꾸지 않습니다. 단지 표현방식을 바꾼 것 뿐입니다.
		그렇기 때문에, Ax를 transposition하려면, AT xT로 바꾸어서 정보가 같도록 유지해줘야하는 것 입니다.
		
		하지만, quadratic form에서는 다릅니다.
		x가 Transposition 되어 A앞에서 matrix multiplication의 규칙에 따라 연산된다면,
		xT는 A에게 다른 영향을 주므로 다른 정보를 담고 있다고 봐야할 것 입니다.
		
		xTA와 xA는 완전 다른 것 입니다.	
		
		의문 1 : Quadratic Equation에서 x^TAx를 계산하는 이유가 무엇일까요?
		
		A라는 matrix를 여기서곱하고 저기서 곱한다는 의미가 있다.
		A의 row를 기점으로 연산하고, 다시 column을 기점으로 연산하고.
		마치 사각형을 한번 굴려서 연산하는 것처럼.
		
		핵심은 우리는 quadratic form을 통해
		각기 다른 A가 가진 변환의 특성을 파악할 수 있다는 점으로 보인다.
```

```yaml
Definitenss 판별 :

vector x가 0이 아닌 경우에,
fx가 output할 수 있는 값을,
function의 함축, 요약을 통해 논리적으로 판가름할 수 있다.

- 예제 1: 단위 행렬 (Identity Matrix)

   • I에 대해 
   x^TIx = ||x||^2이고, 
   x ≠ 0이면 항상 
   ||x||^2 > 0이므로 I는 Positive Definite.

- 예제 2: A = (-1   0)
             ( 0  -2)

   • 이 경우 
   x^TAx = -x_1^2 - 2x_2^2
   가 되어, 
   x ≠ 0이면 항상 음수이므로 A는 Negative Definite입니다.

- 예제 3: A = ( 1  -1)
          (-1   1)

   • 여기서는 x^TAx = x_1^2 - x_2^2가 됩니다. x의 선택에 따라 x^TAx가 양수가 될 수도 음수가 될 수도 있으므로, A는 Indefinite입니다.
```
