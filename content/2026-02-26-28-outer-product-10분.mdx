# 28 Outer Product (10분)

- inner product와의 차이와 계산법
    
    **원래 내가 “Dot Product”라고 알고 있던 “Inner Product”**
    
    `보통 같은 차원의 값 끼리 곱한다음 모두 더하는 연산을 하였었음.` 
    
    `최종값은 Scalar이므로, 정보의 차원을 “축소”한다는 의미가 있음.` 
    
    ![스크린샷 2025-02-10 오전 1.11.02.png](28%20Outer%20Product%20(10%E1%84%87%E1%85%AE%E1%86%AB)%20193029fe18a4803e91cbdf66bebcb5e2/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-02-10_%E1%84%8B%E1%85%A9%E1%84%8C%E1%85%A5%E1%86%AB_1.11.02.png)
    
    **외적 (outer product)은 뭘까?**
    
    ```yaml
    
    외적(Outer Product)은 “abᵀ” 형태로 표기 :
    내적(Inner Product)는 "aᵀb" 형태로 표기 :
    
    	왜 ?? :
    	
    	행렬 계산 규칙때문에.
    	
    		전제 1. 열벡터로 표기
    		전제 2. 행렬 계산 규칙은 각 행과 열을 곱함.
    	
    	설명 : 
    	
    	vector v1 = 열벡터 [a,b]
    	vector v2 = 열벡터 [c,d]
    	
    	그럼 이 둘은 행은 2개, 열은 1개임
    	
    	v1v2 연산시, aᵀ 하면, 행도 1개 열도 1개됨.
    	그래서 inner product는 단 한번만 연산해서 더하면 됨.
    	
    	근데, outer product는 bᵀ로 만들어서 계산시 행도 2개 열도 2개임.
    	
    	그래서 값이 4개 나옴.
    	
    	
    ```
    
    ![스크린샷 2025-03-23 오후 11.10.10.png](28%20Outer%20Product%20(10%E1%84%87%E1%85%AE%E1%86%AB)%20193029fe18a4803e91cbdf66bebcb5e2/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-03-23_%E1%84%8B%E1%85%A9%E1%84%92%E1%85%AE_11.10.10.png)
    
    ```yaml
    해당 연산의 의미 :
    
    하나의 벡터의 각 축의 원소를 연산되는 다른 벡터의 원소만큼씩 증폭시켜서,
    해당 원소를 하나의 독립된 벡터로 만드는 것.
    ```

- triple product
    
    **세 벡터 a, b, c의 곱셈**
    
    세 벡터 a, b, c의 곱셈을 두 가지 방법으로 계산하는 방법
    
    `다른 연산 과정, 같은 값, 다른 연산 효율`
    
    ```yaml
    First Method: (abᵀ)c
    
    outer product로 새로운 matrix를 만든 뒤, matrix - vector product 계산 하는 법.
    
    Second Method: a(bᵀc)
    
    Inner Product (bᵀc)를 계산한 뒤, Vector-Scalar Product를 계산
    ```

- triple product 방법론의 계산 효율성
    
    ```yaml
    1) 첫 번째 방법 (abᵀ)c 의 계산량
    
    		1) 외적(abᵀ) 계산: n × n번의 곱셈
    		   - n×n 크기의 행렬을 만들기 위해 모든 요소 계산 필요
    		   
    		2) 행렬-벡터 곱셈: n × n번의 곱셈
    		   - n×n 행렬과 n차원 벡터의 곱셈
    		
    		총 계산량: O(n²) (n의 제곱에 비례)
    
    2) 두 번째 방법 a(bᵀc) 의 계산량
    
    		1) 내적(bᵀc) 계산: n번의 곱셈
    		   - n개의 요소끼리 곱하고 더하기
    		
    		2) 스칼라-벡터 곱셈: n번의 곱셈
    		   - 하나의 숫자와 n차원 벡터의 곱셈
    		
    		총 계산량: O(n) (2n번의 연산)
    		
    3) 효율성 비교 -
    
    n = 1,000 일 경우
    
    첫 번째 방법: 1,000,000번의 곱셈
    두 번째 방법: 2,000번의 곱셈
    
    n이 커질수록 차이가 커짐
    ```
