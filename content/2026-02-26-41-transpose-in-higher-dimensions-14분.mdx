# 41 Transpose in Higher Dimensions (14분)

- 강의 주제 및 배경
    
    ```yaml
    강의 주제 및 배경 :
    
    주제:
    
    기존의 행렬 전치(transpose) 개념을 확장하여, 2차원(행렬)을 넘어서 
    고차원 텐서(tensor) 에서의 전치(혹은 차원 순서 변경)에 대해 다룸.
    
    배경:
    
    전통적인 선형대수에서는 모든 것이 2차원 행렬(혹은 벡터, 스칼라)로 표현.
    
    하지만 머신러닝과 딥러닝에서는 텐서(3차원 이상, 예 - 이미지, 커널 등)와 같이 고차원 데이터를 다루기 때문에, 
    단순히 행과 열만 전치하는 것 이상의 작업(즉, 여러 차원 간의 순서 변경)이 필요.
    
    ```
    
- 고차원에서의 전치 개념
    
    ```yaml
    텐서 : 3개 이상의 차원을 가진 데이터 구조
    
    일반적인 행/렬 구조에서는 transpose하면 간단하게 차원을 swap 할 수 있었다.
    
    고차원 transpose :
    
    	2d 에서는 :
    	2d에서는 단순 행과 열을 바꾸는 방법밖에 없음.
    		
    		[1 2]
    		[3 4]
    		
    		-> 
    			
    		[1 3]
    		[2 4]
    	
    	행과 열을 바꾼다는 것은 어떤 의미인가?
    	사실 어떻게 해석하느냐에 따라 다를 것.
    	행과 열이 각각 가지고 있는 의미는 우리가 정의한 계산 규칙때문에 생기는 것이기 때문에,
    	어떤 의미를 부여하느냐에 따라 다르지 않을까 생각한다.
    	
    	** 데이터 해석의 관점이 달라질 것.
    	
    	3d 이상의 고차원 transpose :
    	
    	차원 간의 순서를 재배열(permute).
    	차원 순서를 바꿀 수 있음.
    	* 차원 순서를 바꾼다는 것은 계산 순서를 바꾸겠다는 의미.
    	
    	- 여러방식의 차원 순서 재배열 가능
    	ikj, ijk, kij 등...
    	
    	** 아주 고차원적으로 가면 차원의 정보가 바뀌는 것까지 생각해볼 수 있지만..
    	현재의 고차원 transpose는 데이터의 구조를 바꾸는 것으로 이해하여도 괜찮다.
    	
    	** 어떤 library를 활용하느냐에 따라서, 입력해야하는 데이터 구조의 순서가 달라지는 경우들도 있어서
    	주의해야한다.
    
    ```
    
- 프로그래밍에서의 전치
    
    ```yaml
    딥러닝 프레임워크에서는 transpose 함수 제공.
    
    데이터 구조 변경에 활용할 수 있다.
    
    사용법이나 몇 가지 유의할점이 있지만, 이것은 수학에 대한 내용은 아니므로 넘어간다.
    ```
    
- 실제 응용 사례
    
    ```yaml
    ** 딥러닝 프레임워크 사용시 요구하는 데이터구조가 달라 차원 변경이 필요.
    
    4. 실제 응용 사례: 이미지와 CNN 커널
    
    4.1. 이미지의 표현
    
    	이미지 데이터:
    	일반적으로 이미지는 픽셀 값의 배열로 표현되는데, 단순히 행과 열로만 구성된 2D 행렬이 아니라,
    	각 픽셀이 가지는 색 정보(RGB 혹은 RGBA) 때문에 3차원 텐서로 표현됩니다.
    	
    		예: (높이, 너비, 색 채널)
    
    	프레임워크 간 차이:
    	딥러닝 라이브러리에 따라 이미지 텐서의 차원 순서가 다를 수 있습니다.
    	
    		예: PyTorch에서는 보통 (채널, 높이, 너비) 순서, TensorFlow에서는 (높이, 너비, 채널) 순서를 사용합니다.
    		따라서, 프레임워크 간의 호환성을 위해 텐서의 차원 순서를 바꿔주는(전치하는) 작업이 필요합니다.
    
    4.2. CNN 커널(필터)
    	
    	커널의 차원:
    	
    	CNN에서 사용하는 필터(커널)는 입력 채널, 출력 채널, 높이, 너비 등 4차원 텐서로 표현됩니다.
    	라이브러리마다 커널의 차원 순서가 다르기 때문에,
    	예를 들어, 한 라이브러리는 (출력 채널, 입력 채널, 높이, 너비) 순서를 사용하고,
    	다른 라이브러리는 (높이, 너비, 입력 채널, 출력 채널) 순서를 사용할 수 있습니다.
    	
    	전치(차원 재배열)의 필요성:
    	
    	사전에 학습된 모델의 가중치(커널)를 다른 프레임워크에서 사용하려면,
    	해당 커널의 차원 순서를 적절히 전치(재배열)하여 변환해 주어야 합니다.
    	이를 통해 전이 학습(transfer learning) 시에 모델 간 호환성을 맞춥니다.
    ```
    
- 낮은 차원(1D 벡터)에서의 전치
    
    ```yaml
    5. 낮은 차원(1D 벡터)에서의 전치
    
    벡터의 표현:
    	수학에서는 벡터를 행렬(열 벡터 혹은 행 벡터)로 표현하지만, 프로그래밍에서는 보통 1차원 배열로 표현됩니다.
    
    전치 효과:
    	1차원 배열은 단 하나의 차원만 가지므로, 전치 연산을 적용해도 아무런 변화가 없습니다.
    	즉, 1D 객체에서 x.T는 그냥 x와 동일한 결과를 줍니다.
    	
    의의:
    	이로써 “전치”의 개념이 단순히 차원 간 순서 변경이라는 점을 이해할 수 있습니다.
    	2차원 이상의 객체에서는 전치(차원 재배열)가 의미있게 작동하지만, 1차원에서는 변화가 없다는 점을 주의해야 합니다.
    	
    ```
