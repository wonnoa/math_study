# 38 Matrix Transpose (19분)

- matrix transpose
    
    ```yaml
     : "matrix의 row <-> column 을 swap 하는 행위",
     : "ij를 ji한다 라고 받아들일 수 있음."
    ```
    
- 신경망층에서 transpose 활용
    
    ```yaml
    상황 : 입력값 3개를 신경망층 W에 input하여 출력값 2개를 얻고자 하는 상황.
    차원이 축소되어야 한다.
    
    입력값 3개 (x₁, x₂, x₃)
    출력값 2개 (z₁, z₂)
    가중치들 (w₍ᵢⱼ₎ - i는 입력, j는 출력)
    
    W = [w₁₁ w₁₂]
        [w₂₁ w₂₂]
        [w₃₁ w₃₂]
        
    ** W = 변환함수
    
    문제 : 입력값과 W의 차원이 맞지 않음. W에 Column이 3개 필요함.
    해결 : Transpose 사용
    ```
    
- 배치처리에서의 transpose 응용
    
    ![스크린샷 2025-02-10 오후 4.26.16.png](38%20Matrix%20Transpose%20(19%E1%84%87%E1%85%AE%E1%86%AB)%201bd029fe18a480bea847df6dc94ad6b2/%E1%84%89%E1%85%B3%E1%84%8F%E1%85%B3%E1%84%85%E1%85%B5%E1%86%AB%E1%84%89%E1%85%A3%E1%86%BA_2025-02-10_%E1%84%8B%E1%85%A9%E1%84%92%E1%85%AE_4.26.16.png)
    
    ```yaml
    x = nx1
    
    n = 차원의 수 (x,y,z... = 변수의 갯수)
    1 = 정보의 갯수 (벡터 몇개?)
    
    예시 : 3x2를 transpose하면 곱셈이 가능해진다.
    
    W = [w₁₁ w₁₂]   x = [x₁]
        [w₂₁ w₂₂]       [x₂]
        [w₃₁ w₃₂]       [x₃]
     
    
    * 여기서 x는 3차원 단일 정보.
    - w는 2차원 입력에 대한 변환 정보밖에 가지고 있지 않음.
    -> transpose 하여 3차원
    
    의문점 :
    
    transpose하면 같은 정보가 아니잖아.
    원래 2x3인 이유는 x와y, 즉 2개의 차원을 3차원정보로 변환할 수 있는 테이블 이었잖아.
    그런데, 이것을 행과 열을 바꾸면 물론 차원의 수는 맞겠지.
    하지만 의도한 변환정보는 완전히 손실되는 것 아닌가?
    
    그래서 "무엇과 무엇을 곱하는지"에 대한 것이 바뀌면 안될 것 같다는 생각이다.
    
    이해 :
    
    "배치처리하느라 열벡터가 행벡터표기로 바뀌면서, W도 Transpose해주는 것 뿐"
    
    1. 원래 수학 표기는 열벡터
    2. 열벡터를 그대로 사용한다면, W를 있는 그대로 사용해도 된다.
    3. 컴퓨팅에서는 연산 효율 위하여 열벡터를 행벡터로 표현. (세로 벡터를 가로벡터로)
    4. 그러다보니 바로 W를 활용해서 연산 불가능 (차원 정보가 안 맞음)
    5. W를 Transpose해주면 알맞은 차원 multiplication 가능해짐.
    
    이어지는 의문점 : 
    
    "그럼 애초에 열벡터를 matrix로 표시해서 dot product로 연산하면 되지, 왜 굳이 행벡터로 바꾼담?"
    
    gpt 설명은
    	1. 애초에 W가 transpose되어야 하는 상황을 가정함.
    	2. 그런데 배치 처리할 때 열벡터를 나열하느니 행벡터로 바꾸어서 matrix로 만든다함.
    	3. 그럼 W를 transpose하지 않고서도 바로 multiplication할 수 있다고 한다.
    	->> 당연한 소리인데.
    	
    	
    	
    ```
    
- Dimensional Analysis
    
    ```yaml
    정보의 최종 Dimension 크기를 예측.
    
    1. inner dimension이 matching 하는지 확인
    2. outer dimension끼리 곱한 것이 최종 matrix 크기
    
    체크 과정 :
    
    W transpose → 2×3
    W^T × x 가능한지 확인
    
    W^T - 2×3
    x - 3×1
    inner dimension 매칭 (3=3)
    결과는 2×1
    
    되는 원리 :
    	
    	몇 개의 벡터 정보가, 몇 개의 차원을 가진 정보로 변환되었느냐를 보는 것.
    	x = 1개 정보
    	w = 2개 차원 정보로 축소
    	
    	-> 2개 차원가진 1개 정보 = 데이터 크기
    
    하는 이유 :
    
    연산 가능 여부 확인
    결과 행렬 크기 예측
    잠재적 오류 사전 방지
    
    실제 응용: 
    
    신경망에서 Z = W^T × X 계산 시 차원 호환성 검증에 활용
    ```
