# 61 Matrix Decompositions (SVD, QR, LU, Cholesky)

- Matrix Decomposition
    
    ```yaml
    **Matrix Decomposition :**
    
    하나의 복잡한 행렬을 여러 개의 단순한 행렬들의 곱으로 분해하는 과정.
    
    	- 계산 효율성
    	- 고유한 성질 분석
    	- 수치해석과 최적화
    ```
    
- Cholesky Decomposition
    
    ```yaml
    **Cholesky Decomposition :
    
    	분해 형태 :
    		A = L * L^T
    		
    		L   = lower triangular matrix
    		L^T = upper triangular matrix
    	
    
    	조건 :
    		A 가 
    			1) symmetric
    				- 그래야 transpose해도 같은 형태의 L / L^T를 구할 수 있다.
    				
    			2) positive definite
    			
    	왜 나누나 ? :
    		
    		1. Ax = b 를 풀기 쉽다.
    			
    		2. 최적화 문제(예 - 공분산 행렬 최적화)
    		
    		3. 샘플링과 확률분포
    		
    	
    	문제 풀이 예시 :
    	
    	문제:
    			다음 행렬 A에 대한 촐스키 분해를 구하시오.
    			A = [ [4, 2],
    			      [2, 3] ]**
    			
    	해설:
    		1. 가정 :
    		
    		A = L * L^T 로 촐스키 분해할 수 있으며, L은 아래와 같이 하삼각 행렬로 **가정합니다.**
    		    L = [ [l11, 0],
    		          [l21, l22] ]
    	
    		2. L * L^T 식으로 전개 :
    		
    		L * L^T =  [ [l11^2,         l11 * l21],
    	               [l11 * l21,   l21^2 + l22^2] ]
    	 
    	  3. 행렬의 각 원소를 전개한 A의 원소와 비교 :
    	   
    	   - l11^2 = 4  ==>  l11 = 2
    	   - l11*l21 = 2  ==>  l21 = 2 / 2 = 1
    	   - l21^2 + l22^2 = 3  ==>  1 + l22^2 = 3  ==>  l22^2 = 2  ==>  l22 = √2
    	
    		4. 전개 및 검산 :
    			 
    			 따라서, L =    [ [2, 0],
    		                  [1, √2] ]
    		                  
    		   검산: L * L^T = [ [4, 2],
    		                    [2, 1+2] ] = [ [4, 2],
    		                                   [2, 3] ] = A
    	
    			답:
    			촐스키 분해는 
    				
    				L = [ [2, 0], [1, √2] ]
    ```
    
    ```yaml
    왜 positive definite이어야 하나? :
    
    "component를 분해하는 형식 때문. x^2이 matrix의 component값으로 정의됨"
    
    무슨말이냐면 :
    
    	예시 행렬:
    	A = [ [-2,  0],
    	      [  0, -3] ]
    	
    	촐스키 분해(Cholesky Decomposition)는 대칭 양의 정부호 행렬에 대해서만 정의됩니다.
    	그러나 A는 negative definite 행렬이므로 촐스키 분해를 시도하면 다음과 같은 문제가 발생합니다.
    	
    	가정: A = L * L^T 인 하삼각행렬 L을
    	L = [ [l11,  0],
    	      [l21, l22] ]
    	로 두고 진행합니다.
    	
    	L * L^T를 계산하면:
    	L * L^T = [ [l11^2,       l11 * l21],
    	            [l11 * l21,   l21^2 + l22^2] ]
    	
    	A = L * L^T 식의 각 원소를 비교하면:
    	1) l11^2 = -2
    	2) l11 * l21 = 0
    	3) l21^2 + l22^2 = -3
    	
    	여기서 첫 번째 식 l11^2 = -2는 l11이 실수일 경우 불가능합니다.
    	실수의 제곱은 항상 0 이상이어야 하기 때문에, 음수인 -2를 만들 수 없습니다.
    	
    	따라서, negative definite 행렬 A에 대해 촐스키 분해를 시도하면 첫 번째 단계에서 오류가 발생합니다.
    	
    ```
    
- LU Decomposition
    
    ```yaml
    LU Decomposition :
    
    	"Lower triangular * Upper triangular 형태로 decompose 한 것"
    	
    	
    	예시 :
    		A = [ a11  a12  a13 ]
    	      [ a21  a22  a23 ]
    	      [ a31  a32  a33 ]
    		
    	아래의 형태로 나눔 :
    	
    	  [ l11   0    0   ]       [ u11   u12   u13 ]
    L = [ l21  l22   0   ],  U = [  0    u22   u23 ]
        [ l31  l32   l33 ]       [  0     0    u33 ]
    ```
    
    ```yaml
    LU decomposition 하는 법 :
    	"Gaussian Elimination 활용해서, 대각 방향으로 0을 만들어감",
    	"소거를 통해 U 행렬을 만들면서, 사용된 비율(배수)’을 행렬 L 에 저장한다"
    	
    	언어로 표현한 이해 :
    		1. x의 영향력이 지워진 상태의 row 2,3으로 우선 표현.
    		2. y의 영향력이 지워진 상태의 row 3으로 표현.
    		... 이것을 이어감.
    		
    		이 때,
    			pivot : 소거의 기점이 되는 matrix의 시작점.
    			근데 그냥, 영향력을 지울 차원의 첫 정보.. 정도로 이해하면 될 듯.
    			
    			소거에 사용된 배수 :
    			
    			이것이 L의 요소들이 된다.
    			
    	실제 하는 법 :
    	
    		A :
    			[2 4 2]
    			[4 9 8]
    			[2 8 7]
    			
    		1. row 1빼고 모두 1차원 정보 (x)의 영향력 지운다. :
    			
    			row 2 변환 :
    			row 2 - row 1 * 2 = 0 1 4
    			
    			row 3 변환 :
    			row 3 - row 1 * 1 = 0 4 5
    			
    		변환된 시스템 결과 :
    		"결과"
    		"		2 4 2
    				0 1 4
    				0 4 5 "
    					
    					소거에 사용된 배수 :
    						row 2 = 2
    						row 3 = 1
    		
    		2. row 1 / 2빼고 모두 3차원 정보 (y)의 영향력 지운다. :
    			
    			2 4 2
    			0 1 4
    			0 4 5
    			
    			row 3 - row 2 * 4 = 0 0 -11
    			
    		최종 U :
    			
    				2 4 2
    				0 1 4
    				0 0 -11
    			
    					소거에 사용된 배수 :
    						row 3 = 4
    		
    		3. L을 구한다 :
    			
    			"배수를 다 가져오면 됨, 그리고 대각선은 1로 놔둠"
    			
    			최종 L :
    				1 0 0
    				2 1 0
    				1 4 1
    ```

    
    ```yaml
    L과 U를 활용하여 Ax = b 효과적으로 푸는 법 :
    
    	A * x = b  ⟹  L * U * x = b
    	
    	1. 중간 변수 z 도입:
    	
    	"L과 z의 연산 형태로 만들고, z는 U와 x의 연산 결과임"
    	
    		U * x를 z라고 정의합니다. 즉,
    		
    		z = U * x
    		
    		그러면 식은 두 단계로 나뉩니다:
    
    			첫 번째 단계:
    				L * z = b
    
    			두 번째 단계:
    				U * x = z
    	
    	2. 첫 번째 단계: Forward Substitution (L * z = b) :
    	
    	"b값과 L값을 앎으로 z값을 역산하는 법"
    
    		L이 하삼각 행렬이므로, L * z = b의 각 행을 차례대로 풀어 z 벡터의 각 원소를 구할 수 있습니다.
    		예를 들어, 3×3 하삼각 행렬 L과 3×1 벡터 b가 다음과 같이 주어졌다고 합시다.
    		
    		L = [ l11      0      0 ]
    		    [ l21    l22      0 ]
    		    [ l31    l32    l33 ]
    		
    		b = [ b1,
    		      b2,
    		      b3 ]
    
    	이때 L * z = b는 다음의 세 개의 방정식으로 표현됩니다.
    
    		1) 첫 번째 행:  
    		   l11 * z1 = b1  
    		   ⇒ z1 = b1 / l11
    		
    		2) 두 번째 행:  
    		   l21 * z1 + l22 * z2 = b2  
    		   ⇒ z2 = (b2 − l21 * z1) / l22
    		
    		3) 세 번째 행:  
    		   l31 * z1 + l32 * z2 + l33 * z3 = b3  
    		   ⇒ z3 = (b3 − l31 * z1 − l32 * z2) / l33
    
    		"이처럼, 앞에서부터 차례대로(z1, z2, z3) 구하는 방법을 forward substitution이라고 합니다."
    
    		예시 (숫자 예제):
    		
    		L = [ 2   0   0 ]
    		    [ 3   1   0 ]
    		    [ 1   4   2 ]
    		
    		b = [ 4,
    		      8,
    		     10 ]
    		
    		해결 과정:
    		
    		- 첫 번째 행: 2 * z1 = 4  
    		  ⇒ z1 = 4 / 2 = 2
    		
    		- 두 번째 행: 3 * z1 + 1 * z2 = 8  
    		  ⇒ 3*2 + z2 = 8  
    		  ⇒ 6 + z2 = 8  
    		  ⇒ z2 = 8 − 6 = 2
    		
    		- 세 번째 행: 1 * z1 + 4 * z2 + 2 * z3 = 10  
    		  ⇒ 1*2 + 4*2 + 2*z3 = 10  
    		  ⇒ 2 + 8 + 2*z3 = 10  
    		  ⇒ 10 + 2*z3 = 10  
    		  ⇒ z3 = 0
    		
    		따라서 중간 결과 벡터는:
    		z = [ 2,
    		      2,
    		      0 ]
    	
    	3. 두 번째 단계: Backward Substitution (U * x = z) :
    	
    	"z값과 U값을 앎으로 최종 x값을 역산하는 법"
    
    	U가 상삼각 행렬이므로, 마지막 행부터 차례대로 x 벡터의 각 원소를 구할 수 있습니다.
    	예를 들어, U가 3×3 행렬일 때:
    
    		U = [ u11   u12   u13 ]
    		    [  0    u22   u23 ]
    		    [  0     0    u33 ]
    		
    		그리고 이미 구해진 중간 결과 벡터 z가 다음과 같이 주어졌다고 가정합니다:
    		
    		z = [ z1,
    		      z2,
    		      z3 ]
    		
    		그러면 U * x = z는 다음과 같은 세 개의 방정식으로 표현됩니다.
    		
    		1) 세 번째 행:
    		   u33 * x3 = z3
    		   ⇒ x3 = z3 / u33
    		
    		2) 두 번째 행:
    		   u22 * x2 + u23 * x3 = z2
    		   ⇒ x2 = (z2 − u23 * x3) / u22
    		
    		3) 첫 번째 행:
    		   u11 * x1 + u12 * x2 + u13 * x3 = z1
    		   ⇒ x1 = (z1 − u12 * x2 − u13 * x3) / u11
    		
    		이처럼 backward substitution을 통해 마지막 행부터 역순으로 해를 구하게 됩니다.
    		
    		예시:
    		
    		U = [ 3   2   1 ]
    		    [ 0   4   2 ]
    		    [ 0   0   5 ]
    		
    		z = [ 10,
    		      8,
    		      15 ]
    		
    		해결 과정:
    		
    		- 세 번째 행: 5 * x3 = 15
    		  ⇒ x3 = 15 / 5 = 3
    		
    		- 두 번째 행: 4 * x2 + 2 * x3 = 8
    		  ⇒ 4 * x2 + 2 * 3 = 8
    		  ⇒ 4 * x2 + 6 = 8
    		  ⇒ 4 * x2 = 2
    		  ⇒ x2 = 2 / 4 = 0.5
    		
    		- 첫 번째 행: 3 * x1 + 2 * x2 + 1 * x3 = 10
    		  ⇒ 3 * x1 + 2 * 0.5 + 1 * 3 = 10
    		  ⇒ 3 * x1 + 1 + 3 = 10
    		  ⇒ 3 * x1 + 4 = 10
    		  ⇒ 3 * x1 = 6
    		  ⇒ x1 = 6 / 3 = 2
    		
    		따라서 최종 해는:
    		x = [ 2,
    		      0.5,
    		      3 ]
    ```
    
    ```yaml
    왜 나눔 ? :
    
    왜 굳이 LU 분해를 할까?
    
    선형방정식(Ax = b) 풀기 :
    
    A가 n×n 정사각 행렬일 때, A x = b의 해를 효율적으로 구하고 싶다면, A를 “L과 U의 곱”으로 만들어두면 좋습니다.
    왜냐하면,
    먼저 Lz = b를 풀고(z를 구함),
    다음 Ux = z를 풀면,
    최종적으로 x를 얻을 수 있습니다.
    이 과정은 직접 A^(-1)(A의 역행렬)을 구하는 것보다 계산량이 훨씬 적고, 수치적으로도 안정적입니다.
    
    행렬식(Determinant) 계산 :
    
    LU 분해를 해두면, 행렬 A의 행렬식(det(A))을 “하삼각 행렬의 대각 원소 곱 × 상삼각 행렬의 대각 원소 곱”으로 빠르게 구할 수 있습니다.
    즉, det(A) = (l11 × l22 × … × lnn) × (u11 × u22 × … × unn).
    이것은 큰 행렬의 행렬식을 직접 구하는 방법보다 훨씬 간단합니다.
    
    행렬 연산 최적화 :
    
    A^(-1)을 구하거나, 어떤 반복적인 선형시스템을 풀 때(L이 바뀌지 않는다고 할 때) 한 번 LU 분해를 해두면 여러모로 재사용이 가능해요.
    예: 다양한 b에 대해 A x = b를 반복해서 풀 때, LU 분해가 있으면 매번 L과 U에 대해서만 전진대입, 후진대입을 수행하면 되니까 빠르고 안정적입니다.
    ```
    
    ```yaml
    “반드시” LU 분해가 되는가? :
    
    사실 모든 정사각 행렬 A가 무조건 A = L * U 형태로 깔끔하게 나눠지는 것은 아닙니다.
    어떤 문제?
    A가 연산 과정 중에 특정 피벗 요소가 0이 되어서 나눗셈이 불가능한 상태가 생길 수 있음.
    이런 상황을 처리하기 위해 Pivoting 기법(행이나 열을 바꿔주는 과정)을 쓰는 경우가 흔합니다.
    즉, “PA = L * U” 형태(P는 Permutation 행렬)로 분해하기도 합니다.
    대부분의 프로그래밍 라이브러리(NumPy, MATLAB 등)에서 “LU 분해”라고 하면 내부적으로 피벗팅을 고려해서 “PA = LU”를 구해줍니다.
    ```
    
- QR Decomposition
    
    ```yaml
    1. QR 분해란 무엇인가요?
    --------------------------
    정의:
      - 어떤 실수 행렬 A를 “정규직교(orthogonal) 행렬 Q”와 “상삼각(upper triangular) 행렬 R”의 곱으로 나타내는 것을 QR 분해라 부릅니다.
      - 수식으로는 A = Q × R라고 표현합니다.
      - 여기서 Q는 직교 행렬이므로 Qᵀ = Q⁻¹ (복소수의 경우 Q* = Q⁻¹)이며, R은 상삼각 행렬입니다.
    
    정사각 행렬이 아닌 경우:
      - A가 m×n (m ≥ n) 직사각 행렬이어도 QR 분해를 확장해서 사용할 수 있습니다.
      - 이때 Q는 m×m 정사각 행렬(직교)이고, R은 m×n 형태로, 아래쪽 일부가 0 블록을 가지는 구조가 됩니다.
    
    2. 왜 QR 분해가 필요할까요?
    --------------------------
    직교 행렬의 장점:
      - 직교 행렬 Q는 벡터의 길이와 각도를 보존하는 회전(혹은 반사) 연산으로 해석됩니다.
      - ∥Qx∥ = ∥x∥ (즉, 벡터의 노름이 유지됨).
      - 계산상 Qᵀ를 사용하면 Q⁻¹를 직접 구할 필요가 있어 효율적입니다.
    
    최소제곱 문제 (Least Squares):
      - 선형 회귀나 데이터 피팅 문제에서, min ∥Ax − b∥ 같은 문제를 풀 때 QR 분해가 자주 사용됩니다.
      - A = Q × R로 표현하면 ∥Ax − b∥ 문제를 ∥QRx − b∥로 바꿀 수 있고, 직교 변환 Q 덕분에 식이 단순해져 상삼각 행렬 R을 빠르게 풀 수 있습니다.
    
    수치 안정성 (Numerical Stability):
      - QR 분해는 벡터 길이를 보존하는 직교 변환을 사용하므로, 오차 전파가 상대적으로 적어 수치적으로 안정적인 결과를 제공합니다.
    
    직교기저 (Orthonormal Basis) 구하기:
      - Gram–Schmidt 과정을 통해 행렬 A의 열 벡터들을 정규직교화할 때 자연스럽게 QR 분해가 등장합니다.
      - 이 과정에서 얻은 정규직교 벡터들이 Q를 구성하고, 그 계수들이 R에 들어갑니다.
    
    3. 간단한 예시로 이해하기
    --------------------------
    예를 들어, A가 3×2 직사각 행렬이라고 가정합니다:
    
        A = [ 1   1 ]
            [ 1   2 ]
            [ 1   3 ]
        (즉, 3행 2열의 행렬; 3차원 공간에 두 개의 열 벡터)
    
    QR 분해의 목표:
      - A를 A = Q × R 형태로 분해합니다.
      - 여기서 Q는 3×3 직교 행렬이고, R은 3×2 행렬이지만 실제로는 상단의 2×2 상삼각 블록과 나머지 0 블록으로 구성됩니다.
    
    3.1. Gram–Schmidt 알고리즘 관점:
      - A의 두 열을 각각 a₁, a₂라고 합시다.
        * a₁ = (1, 1, 1)ᵀ
        * a₂ = (1, 2, 3)ᵀ
    
      Step 1:
        - q₁ = a₁ / ∥a₁∥
        - ∥a₁∥ = √(1² + 1² + 1²) = √3
        - 따라서, q₁ = (1/√3) × (1, 1, 1)ᵀ
    
      Step 2:
        - 먼저, a₂에서 q₁ 방향 성분을 제거합니다.
        - u₂ = a₂ − (q₁ᵀ a₂) · q₁
        - q₁ᵀ a₂ = (1/√3) × (1 + 2 + 3) = 6/√3 = 2√3
        - 따라서, u₂ = (1, 2, 3)ᵀ − 2√3 × q₁
                 = (1, 2, 3)ᵀ − 2 × (1, 1, 1)ᵀ
                 = (−1, 0, 1)ᵀ
        - ∥u₂∥ = √((-1)² + 0² + 1²) = √2
        - q₂ = u₂ / ∥u₂∥ = (1/√2) × (−1, 0, 1)ᵀ
    
      - 이제 q₁와 q₂는 서로 직교하며, 길이가 1인 정규화된 벡터들입니다.
      - 만약 열이 3개라면, q₃도 같은 방식으로 구할 수 있습니다.
      - Q 행렬는 보통 [ q₁  q₂  (q₃, 있을 경우) ] 형태로 구성됩니다.
        * 2열짜리 경우, 3×2 형태의 Q 블록으로 사용하거나, 3×3 정사각 Q로 확장할 수 있습니다.
    
      - R 행렬은 “A의 열을 Q의 직교기저에 대해 표현한 좌표”를 나타냅니다.
        * a₁ = r₁₁ · q₁  →  r₁₁ = ∥a₁∥ = √3
        * a₂ = r₁₂ · q₁ + r₂₂ · q₂  →  r₁₂ = q₁ᵀ a₂ = 2√3,  r₂₂ = q₂ᵀ a₂ = 2
    
        - 따라서, R = [ r₁₁  r₁₂ ]
                       [  0    r₂₂ ]
          (나머지 아래 행은 0으로 채워집니다.)
    
    4. QR 분해의 의미 — “회전(직교변환)과 상삼각 스케일링”
    -----------------------------------------------------------
      - Q: 벡터 공간을 회전 또는 반사하는 역할을 하여, 좌표축을 재정렬하면서 벡터의 길이(노름)를 보존합니다.
      - R: 상삼각 행렬로, 각 좌표축이 얼마나 스케일(늘어나거나 줄어드는지)을 나타냅니다.
    
    5. 실제 활용
    --------------------------
      - 최소제곱(Least Squares) 문제:
          * “Ax ≈ b” 문제에서 A를 QR 분해하면, 문제를 ∥Rx − Qᵀb∥ 형태로 단순화할 수 있습니다.
      - 정사각 행렬의 역 계산:
          * 만약 A가 정사각이고 A = Q × R이면, A⁻¹ = R⁻¹ × Qᵀ로 계산할 수 있습니다.
      - 직교기저 생성:
          * Gram–Schmidt 과정을 통해 A의 열 벡터들을 정규직교화할 때 QR 분해가 자연스럽게 등장합니다.
      - 수치 안정성:
          * LU 분해보다 특정 상황에서 더 안정적인 결과를 제공할 수 있습니다.
    
    6. 결론
    --------------------------
      - QR Decomposition은 다음과 같은 두 가지 단계로 볼 수 있습니다:
          (1) 직교(또는 유니터리) 변환 Q를 사용하여 행렬 A를 회전 또는 반사시키고,
          (2) 상삼각 행렬 R을 통해 각 좌표축에 대해 스케일링하는 방식.
      - 이 방식은 복잡한 행렬 A를 보다 단순한 구성요소로 분해하여, 최소제곱 문제, 직교기저 찾기, 신호 처리, 통계, 머신러닝 등 다양한 분야에서 널리 사용됩니다.
      - 특히, Least Squares 문
    ```

    ```yaml
    2. SVD란 무엇인가? :
    
    	"3개의 matrix들의 multiplication으로 분해함",
    	
    	A = U * Sigma * V^T
    	
    		회전,
    		스케일링,
    		다시 회전
    				
    각 component 설명 :
    
    U: m×m orthogonal matrix (복소수라면 유니터리)
    
    	orthogonal matrix :
    		"matrix내의 vector들이 orthogonal함"
    			- 각 vector의 norm (길이) = 1
    			- 각 vector들 끼리의 dot product = 0
    		
    Sigma(Σ):
    
    	대각선에 존재하는 값(특이값)
    		- matrix의 주어진 방향에 대한 스케일링 정보를 나타냄
    		- 왜 대각선, 해당 차원으로에 대한 스케일링값 이니까
    
    V: n×n 직교(또는 유니터리) 행렬이며, 식에서는 전치(V^T) 형태로 나타남
    
    		주요 개념 :
    		
    		특이값(singular values): Sigma의 대각 성분들이며, 보통 크기가 큰 순서대로 배열합니다(σ1 ≥ σ2 ≥ … ≥ 0).
    		
    		‘직교/유니터리’ 행렬이란? :
    		
    		직교 행렬: 2차원(실수) 회전·반사를 나타내며, Q^T = Q^-1인 행렬.
    		유니터리 행렬: 복소수 버전의 직교 행렬. U^* = U^-1.
    			(행렬 U가 유니터리(unitary)라는 것은, U의 켤레 전치(즉, U*; 복소수 성분에 대해 전치를 취하고 각 성분의 복소켤레를 취한 행렬)와 U를 곱하면 단위 행렬 I가 된다는 의미)
    		
    		SVD에서도 이 “직교/유니터리” 성질이 매우 중요합니다. 왜냐하면, 길이와 각도를 유지하면서 벡터를 돌리거나(회전, 반사) 옮기기 때문입니다.
    		
    		** U와 V^T는 아예 다른 matrix이다
    ```
