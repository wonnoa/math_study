# 33 Identity Matrix (6분)

- Identity Matrix의 정의와 의미
    
    Identity Matrix의 정의
    
    ```yaml
    기본 특성 :
    
    	표기법 : I
    	크기 : n x n 크기의 정사각 matrix
    	규칙 : 정사각형 matrix에서 row number 와 column number가 같을 때, 값이 1
    	
    				i와 j가 같을 때 1, 나머지 0
    	
    	예시 :			
    				I(3×3) = 
    										[1 0 0]
    						        [0 1 0]
    						        [0 0 1]
    ```
    
    Identity Matrix의 의미
    
    ```yaml
    일반 숫자에서 1은, 그것에 1을 곱하면 그 수가 나온다는 것.
    
    마찬가지로, Identity Matrix는 matrix에 그것을 곱하면 그 matrix가 나온다.
    
    A x I = A
    
    이해한 원리 :
    
    	I의 의미는 "각 정보열에 할당된 차원 정보를 단 한번만 세는 것"이다.
    	그리고 정사각형으로 확장되므로, row자체가 corresponding하는 차원 정보를 가지고있다.
    
    	 변환A 1차 2차 3차 용 변환       I
    			  [1  2  3]              [1  0  0] 1차원 (x)
    			  [4  5  6]              [0  1  0] 2차원 (y)
    			  [7  8  9]              [0  0  1] 3차원 (z)
    		
    		-> 1차원에 대한 벡터 정보만 가져오면 1x,5y,7z
    		-> 2차원에 대한 벡터 정보만 가져오면 2x,5y,8z
    		-> 3차원에 대한 벡터 정보만 가져오면 3x,8y,9z
    	
    		이 정보를 다 가져와서 더하면 'A' 그 자체를 다 더한것과 같다.
    	
    		그 외, 각 정보 열 1,2,3에게 할당된 1,2,3차원의 정보 외에 어떠한 다른 정보도 "count하지 않음"
    		
    		예를 들어, I의 1열은, 변환 A에서 1차원에 할당된 정보를 가져온다. 만약 I의 2차원, 3차원 정보도 존재했다면
    		추가적으로 A에서 2차원, 3차원에 해당하는 정보를 가져왔겠으나, 0이므로 1차원 정보만 가져온다.
    	
    		결론은, I는 matrix의 정보 그 자체만 가져오도록 정교하게 설계되어 있는 표이다.
    ```
    
    예시로 증명
    
    ```yaml
    Matrix A:    Identity Matrix:
    
    [3 4]  ×  [1 0]
    [5 6]     [0 1]
    
    1단계 - 결과의 각 위치 값을 구함 :
    
    row 1, column 1: [3 4] · [1] = 3×1 + 4×0 = 3
                             [0]
    
    row 1, column 2: [3 4] · [0] = 3×0 + 4×1 = 4
                             [1]
    
    row 2, column 1: [5 6] · [1] = 5×1 + 6×0 = 5
                             [0]
    
    row 2, column 2: [5 6] · [0] = 5×0 + 6×1 = 6
                             [1]
                        
    2단계 - 결과를 모으면 :
    
    여전히 같은 Matrix
    [3 4]
    [5 6]
    ```
    
    정보 변환 차원에서 증명 (Matrix Multiplication의 원리를 이해한 측면에서)
    
    ```yaml
    변환 규칙이 그렇게 되어있기 때문에, 위 처럼 나올 수 밖에 없다.
    
    1차원용  2차원용   정보 1  정보 2
    [3       4]  ×  [1       0]  <- 1차원
    [5       6]     [0       1]  <- 2차원
    
    정보 1을 변환기에 넣으면,
    1차원용 정보가 그대로 나옴, 2차원 정보는 존재하지 않음.
    
    정보 2를 변환기에 넣으면,
    2차원용 정보가 그대로 나옴, 1차원 정보는 존재하지 않음.
    
    왜 그러냐?
    
    규칙 자체를 그렇게 설계해놓았기 때문.
    
    가로 1은 세로 1에 대응되고, 대응 되는 차원 정보의 1만 셈한다.
    가로 2는 세로 2에 대응되고, 이것은 새로운 2차원이며, 해당 정보에 대하여 1만 셈한다.
    따라서, 넣은대로 나온다.
    
    Identity Matrix가 가로로 진행될 때마다 (정보가 하나씩 추가될 때마다),
    세로의 차원 정보가 세로로 하나씩 추가 된다.
    그리고 해당 차원에 대한 정보만 한개 변환하여
    다시 그 정보를 그대로 뱉어내도록 설계되어있다.
    
    핵심은,
    
    가로로 정보가 늘어날 때마다, 세로로 차원이 추가되고,
    가로 차원과 세로차원의 끝단인 마지막단에 1이라는 정보를 새롭게 추가.
    끝단이기 때문에 대각선으로 생성될 수 밖에 없음.
    왜 끝단이냐면, 하나가 늘어났고 또 아래로도 늘어났고, 늘어난 것을 공통적으로 반영할 수 있는
    칸이 끝단밖에 없기 때문.
    [ 0 0 0 ]
    [ 0 0 0 ]
    [ 0 0 0 ]
    
    여기서 정보만 하나 늘린다면
    아래처럼되고, 그럼 해당 정보가 늘어난 것을 표기할 수 있는 곳이
    abc인데
    [ 0 0 0 a]
    [ 0 0 0 b]
    [ 0 0 0 c]
    
    여기서 다시 차원도 늘리면
    [ 0 0 0 a]
    [ 0 0 0 b]
    [ 0 0 0 c]
    [ x y z d]
    
    이고,
    차원도 늘었고 정보도 늘었다고 공통적으로 표기할 수 있는 칸은 d밖에 없음.
    
    내 생각에는 그냥,
    이 identity matrix는 이렇게 말하는 것 같다.
    
    내가 변환을 위해 input할 정보를 하나 늘릴꺼고, 거기에 따라서 차원도 하나 늘릴껀데,
    input할 정보는 늘어난 차원에 대한 정보만 1개.
    
    그리고 이 1이라는 정보는 해당 차원에대한 정보 1개 를 세는 단위.
    
    정리하면 :
    "내가 새로운 정보를 변환기에 넣을 건데, 이 새로운 정보는 새롭게 추가된 차원에 대한 정보이고,
    그 차원에 대한 정보 1만 변환기에 넣을게" 이다
    
    ```
