# 60 The Rank of a Matrix

- Rank
    
    ```yaml
    definition : 
    	"matrix의 row / column 중에서 linearly independent한 벡터들의 최대 갯수."
    	
    	만약, matrix A의 rank가 r이라면, A의 행 또는 열 중, r개의 벡터는 서로 선형 독립이다.
    	나머지 벡터들은 r개의 선형 결합으로 표현될 수 있음.
    
    해석 :
    		벡터들로 표현 가능한 차원의 수 (혹은 독립적인 정보)
    		
    		
    		matrix에 5개의 vector가있다 하여도, 
    		rank가 3이라면, 해당 matrix로 표현 가능한 차원은 3차원.
    		
    		개별적인, 겹치지 않는 관계 정보를 나타낼 수 있는 수를 말함.
    ```

- linear independence and applications
    
    ```yaml
    이해 :
    	
    	"새로운 차원, 즉 새로운 변수에 대한 정보를 말해줄 수 있는가를 확인한다"
    
    application 1. feature selection :
    
    	특정한 feature가 다른 feature들의 조합으로 표현 가능하다면, 혹은 거의 같은 정보를 준다면 
    	불필요할 수 있다.
    	
    	예 : 
    		feature 1 - yearly age
    		feature 2 - hourly age
    			
    			yearly age = hourly age x constant
    			
    			meaning. does not reveal new relationship btw age and other factor.
    			
    			만약 과학에서,
    			신체 노화도와 시간의 관계를 밝히는데, yearly age 나 hourly age 나
    			노화도와 age의 관계식 자체는 같다는 것이겠지.
    			
    application 2. dimensionality reduction :
    
    	SVD를 활용, 독립적인 축을 찾는다
    
    application 3. data interperetation :
    	
    	서로 다른 독립적 특성들을 가지고 있는 데이터들과,
    	다른 특성들로 표현 가능한 데이터들을 구분가능.
    	
    	
    간단 예시 :
    	
    	matrix
    		feature 1. 키
    		feature 2. 몸무게
    		feature 3. 키 + 몸무게
    		
    		키 + 몸무게는 feature 1과 3의 linear combination으로 표현 가능.
    		hence, feature 3는 linearly dependent, matrix는 rank = 2		
    	
    
    ```
    
- how to compute matrix rank
    
    ```yaml
    method 1. gaussian elimination :
    
    	non-zero row의 갯수를 세면 그것이 rank.
    
    1. 	
    	⎡ 1  2  1  3 ⎤
    	⎢ 2  4  0  2 ⎥
    	⎢ 3  6  1  5 ⎥
    2. 
    	⎡ 1  2  1  3 ⎤
    	⎢ 0  0 -2 -4 ⎥
    	⎢ 0  0 -2 -4 ⎥
    
    3. finally rank = 2
    	⎡ 1  2  1  3 ⎤
    	⎢ 0  0 -2 -4 ⎥
    	⎢ 0  0  0  0 ⎥
    	
    		why ? :
    		
    			하나의 equation (vector)가 0,0,0,0이 되어서 최종적으로 "소거"될 수 있다면,
    			다른 equation으로 표현될 수 있음을 의미한다.
    			
    			소거되지 않는, 독립적인 정보를 포함하는 equation (변수들간의 관계식)이기 때문.			
    			
    		주의 사항 :
    		
    			matrix가 커지면, elimination 연산과정에서 오차가 발생하여 rank 잘못 계산 가능.
    ```
    
    ```yaml
    method 2. singular value decomposition (SVD) :
    
    A = *UΣV^T*
    		
    		A = m x n matrix
    	**	U = m x m orthogonal matrix
    		V = n x n orthogonal matrix
    		Σ = m×n 형태의 대각선(diagonal-like) 행렬, singular value들이 대각 성분으로 들어있음
    	
    	*Σ의 대각선에 놓이는 “특이값(singular values)” 중 0이 아닌 것의 개수가 곧 행렬의 랭크입니다.*
    
    *** "구체적 알고리즘은 복잡하다 함"
    ```
    
- properties of matrix rank
    
    ```yaml
    행(row)과 열(column)의 선형 독립(linear independence) 요소의 수는 항상 동일.
    
    1) Matrix Rank :
    
    	"언제나 matrix의 m과 n중 작은 것의 이하 숫자이다."
    	
    	3x2이거나 2x3일 때,
    		
    		3차원을 데이터 두 개만으로 표현할 수 없지.
    		2차원을 데이터 3개로 표현하면 하나는 잉여이고.
    		
    2) Full Rank : 
    
    	rank(A) = min(m,n)일 때.
    	
    
    3) Rank Deficient :
    
    	full rank가 아닌경우
    	종속된 정보가 있는 경우
    
    4) Zero Matrix / Rank 0 :
    
    	zero matrix = 모든 원소가 0
    	rank 0 = 오로지 zero matrix만이 zero rank가 될 수 있다.
    
    5) Square Matrix와 Invertibility :
    
    	square matrix A가 invertible 하려면, rank(A) = n이어야 한다.
    	rank(A)가 n이 아닌경우, 적어도 하나의 row나 column은 종속적인 것으로 이해할 수 있다.
    	이것은 det가 0이 아니다라는 조건과 같은 것으로 이해할 수 있다.
    	종속된 정보가 존재한다는 것은, 역연산하는데 필요한 정보가 유일하지 않다는 것이다.
    		(singularity와 혼동하여서는 안 된다, 오히려 반대이다)
    ```
    
    ```yaml
    Square Matrix와 Invertibility :
    
    square matrix의 rank(A)가 n미만일 경우 invertible 하지 않다.
    	
    	간단 설명 :
    		"1. 하나의 벡터 데이터가 dependent하기 때문에 rank(A)가 n미만인 것임",
    		"2. 이 경우 해당 matrix로 변환 연산을 감행하면 하나의 output set에 대하여 여러개의 input set가능",
    		"3. 변환 연산을 되돌릴 수 있는 절대 단일의 A-1 도출 불가",
    					
    	mechanism :
    		
    		개념 :
    		
    			null space :
    			
    				Ax = 0 를 만족하는 x들의 set
    				
    				0이 아닌 벡터 d가 Ad = 0을 만족하면, d는 A의 영공간에 속한다.
    		
    		AI 설명 :
    			Null space와 singularity of matrix :
    				
    				null space가 있다면, 0벡터가 아닌 경우에도 matrix 변환값이 0가 될 수 있음.
    				따라서, 하나의 row나 column이 다른 것에 linearly dependent함.
    				
    				** linear independence의 정의에 어긋남.
    			
    		한 출력에 대하여 여러 입력이 가능해지므로 inverse한 값을 찾을 수 없다 :
    			
    			0이 아닌 벡터 d가 Ad = 0을 만족하면, d는 A의 영공간에 속하면
    			
    			A(x+d) = Ax + Ad = Ax + 0 = Ax 
    				가 성립.	
    		
    			x와 x+d는 다른데, 같은 출력 Ax를 만들어냄
    			
    			** 일방향 논리적인 설명이나, 종합적인 설명은 아닌 것 같다.
    			
    			
    		이해 시도 :
    		
    			"다만 이 이해는, 상수배 equation관점에서 보았으므로, 한계가 있는 설명임을 알게되었다"
    			
    			1. 만약 input이 다른데 output이 같을 경우 :
    				"이 변환 현상을 되돌릴 수 있는 단 하나의 방법론이 존재하지 않는다",
    				"추론할 근거가 유일하지 않기 때문"
    				
    				예시 :
    					4x = 8 이라는 수식이 존재
    					/4 하면 4의 효과를 되돌릴 수 있으며, x를 추론해낼 수 있다.
    					이것이 4에 대한 /4의 역연산인자이다.
    					그리고 이것이 유일의 인자이다. (/2 /2 도 되긴함, 그렇게 치면 matrix도 여러개면 되긴하지)
    			
    				그런데 만약 이걸 switch 한다면 :
    					
    					수식 x에 대하여 4라는 수를 넣을 때, 8이라는 수식관점에서 보자.
    					input이 4라는 관점이다.
    					
    					4x = 8, 
    					2x = 8, 
    					8x = 8
    					뭘 넣어도 8이 나온다고 치자.
    					
    					수식 x의 효과를 되돌릴 수 있는 단 하나의 방법이 존재하지 않게 된다.
    					마치 블랙홀처럼..
    			
    			2. Rank(A)가 n보다 작은 Square Matrix의 경우 :
    				
    				"하나의 행이나 열이 dependent하다, 결국 같은 equation이라는 뜻"
    				
    				Matrix를 변환 equation의 관점에서 본다면,
    
    				A = [a  b]   [x]    = output 1
    						[c  d]   [y]    = output 2
    				
    				row ab와 row cd는 결국 같은 관계식이다.
    				(하나가 다른 하나의 상수배라고 치면)
    				(벡터 관점으로 보아도 x,y의 관계를 나타내는 데이터포인트들이다)
    				
    				이 system of equation에서 우리는 해당 system을 만족시킬 수 있는
    				유일한 [x,y] set을 찾고 있는 중이다.
    				
    				하지만 무수한 x,y set이 존재한다.
    			
    			3. Square Matrix를 Equation관점에서 보는 것 :
    			
    				"각 equation들이 본질적으로 같으므로, system 성립 가능한 x,y의 set이 무한하거나 없다"
    				
    				output 1,2에 대하여 matrix A를 만족시킨
    				vector set (x,y)가 바로 transform 되기전의 input vector이다.
    				하지만 하나의 equation이 다른 equation의 상수배이므로,
    				가능한 set(x,y)는 무한하다고 하였다. (output 1,2도 상수배일 경우)
    				
    				그렇다면 output을 근거로 단 하나의 x,y를 도출해내는 A-1을 얻는 것은 불가능하다.
    			
    				예시 :
    					
    				A
    					2 4 x 6
    					4 8	y	12
    				
    				가능한 set
    					set 1 (1,1)
    					set 2 (2,1/2)
    					set 3 ...
    					
    				(6,12)를 원래의 x,y로 되돌릴 수 있는 단 하나의 규칙은 무한하다.
    				그렇다면 A-1을 기술하는 것은 불가능하다.
    				(가능할지도, 무한대라는 기호를 쓴다면.. 여튼...)
    					
    			4. 왜 이런일이 벌어질까? :
    			
    				eq 1과 eq 2가 아무런 특별한 관계 규칙이 없어서 일지도.
    				만약 둘이 상수배가 아니라면, "딱 떨어지는" 하나의 규칙
    				singular한 규칙이 있을텐데, 뭘 해도 되는 상황이기에 그런 듯 하다...
    				아마
    				linear combination으로 다른 벡터를 표기할 수 있는 경우에도 그럴 듯 싶다.
    				system을 만족시킬 수 있는 set(x,y)가 여럿 있을 것 같다는 직감이 있다.
    				
    			5. 아직 더 이해해야하는 것 :
    				
    				왜 어렵게 null space관점으로 설명하는가
    		
    			6. 깨달은 것 :
    				
    				1. 
    				ab - cd는 결국 "종속성"과 관련 있었나보다.
    				그럴 것이라고 짐작은 했는데, 구체적인 메카니즘은 모르겠구먼.
    				
    				2.
    				어려운 문제 푸는법 :
    				컴퓨터 화면보면서 working memory로만 이해하려고 하지말고
    				써보면서 뇌를 활성화 시키면서 집중을 유지하면서 논리적 연결성을 이어나가는 것.
    				그러다보니 equation 관점에서 문제를 바라볼 수 있는 아이디어가 떠올랐다.
    				구체적으로 이해하는 "느낌"이 들었기 때문.
    				
    				
    				
    								
    				
    ```
