```yaml
- What is Matrix
    
    정의 :
    ***

    해석 :

    matrix는 데이터 그 자체가 될수도 / 데이터를 변환하는 함수가 될수도 (사용하기나름) :
    
    1. 데이터 그 자체가 되는 경우
    
    # 학생들의 과목별 점수
        v1  v2  v3
       [90  88  95]  수학점수 벡터
       [85  92  89]  영어점수 벡터
       [95  87  93]  과학점수 벡터
    
    2. 변환 함수가 되는 경우
    
    # 뭐.. 위 점수를 과목마다 가중치를 주는 변환법을 쓴다.. 
    # 라고 하면 하나의 matrix가 곱해질 수 있겠네
    
      [0.5  0   0  ]  # 수학 50%
      [0    0.3 0  ]  # 영어 30%
      [0    0   0.2]  # 과학 20%
    
    3. 변환 한다고 하면
    
    	[90 88 95]   [0.5 0   0  ]
    	[85 92 89] × [0   0.3 0  ] 
    	[95 87 93]   [0   0   0.2]
    ```

- 선형성의 정의
    
    ```yaml
    선형성(Linearity)의 정의 :
    
    두 벡터 X, Y와 스칼라(상수) a, b에 대해 변환 T가 선형이면,
    
    T(aX + bY) = a T(X) + b T(Y)
    
    왜 선형성 ? :
    
    "곱하고 더하든, 더하고 곱하든, 같은 비율로 숫자들을 더한다는 것을 선형대수학은 피할 수 없다"
    ```
    
- Linear 아닌 경우
    
    ```yaml
    아닌 경우 :
    
    T(x, y) = (x², y²)
    와 같이 각 성분을 제곱하는 변환은 선형성이 깨집니다.
    왜냐하면, T(2(x, y)) = (4x², 4y²)인데, 이는 2T(x, y) = (2x², 2y²)와 다르기 때문입니다.
    
    ->> 왜 아닌가 ? :
    
    제곱의 성질.
    제곱이란 각 변수를 자기의 수 만큼 '곱'하는 것.
    여기서 선형성이 깨진다.
    x,y 증가 비율 관계에서 x와 y의 증가비율을 다르게 잡았다.
    ```
