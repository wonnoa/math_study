# 62 Rank After Multplication

```yaml
: "A와 B의 Multiplication의 결과 matrix의 rank는 언제나 둘 중의 min rank를 따름"

	rank(AB) ≤ min(rank(A), rank(B))

설명 :
	
	1. Matrix A와 B를 연산
	2. AB의 rank는 바뀔까?
	3. 안 바뀐다. A나 B의 rank의 둘중 작은 것의 이하가된다.
	
왜 그렇게 되나 :
	
	1. AB는 A와 B의 linear combination의 합성
	2. AB의 모든 벡터는 A와 B의 linear combination으로 표현 가능함
	3. 새로운 독립된 방향의 벡터 만들어낼 수 없음.
	
	Case 1. A (고차원) x B (저차원) = 저차원데이터가 고차원으로 표현되지만 고차원의 모든 데이터를 표현하기에 수가 모자람.
	Case 2. B (저차원) x A (고차원) = 고차원데이터가 저차원으로 표현됨
	
		-> case 1의 경우 데이터 차원은 큰게 맞은데, 표현력 (Rank)가 적음
		-> case 2의 경우 고차원 데이터는 저차원으로 표현됨
	
	하지만 본질은,
		AB는 A와 B벡터들의 linear combination이므로, 기존 가지고 있던 최소 rank를 벗어날 수 없다는 것.
		

선형 결합으로 표현 가능하여 Rank는 줄어든다는 것 :

	A =       B =          AB =
		[2 1]      [2]          [9]
		[4 5]      [5]          [33]

	A의 Rank는 2 였다.
	B와 연산한 뒤 1로 줄어들었다.
	Rank를 2로 유지할 수는 없다.
	
	더 낮은 차원인 B와 연산하였음.
	B [2,5]가 독립적인 vector이고, AB가 독립적인 Vector B와 종속 관계임을 증명하면,
	말이 될 것이다.	

```

```yaml
의문 :
"벡터끼리 곱하고 더해서 결합되는 과정에서, 어떻게든 회전이 되거나 하면서 새로운 차원 정보가 나올 수 없나?"

	의문 설명 :
		
		조건 :
			A * B
			
			1. B = 3 by 3 matrix
			2. rank(A) = 2
			3. row 1과 2의 simple addition으로 row 3이 탄생한 경우의 dependency임
			
		-> 복잡한 matrix 연산 과정에서 row3이 새로운 rank를 얻게 되는 경우의 수가 없는가?
		
		**상수배
		
		
		아주쉬운 기하학적인 직관적 이해 :
			matrix A = 공간을 같은 비율로 통째로 변화시킴
			B의 모든 벡터 "열"은 같은 비율로 변화되어 사실상 같은 공간내에 있는 것과 같음.
			1과 2의 addition이 여전히 "새로운 방향"을 제시하지 못할 것임이 시각적 상상력으로는 어느정도 당연함.
			
			1. 그러나 이것이 실제라고 장담할 수 없으며, 연산 과정의 조작으로 인하여 다른 방향성을 갖게 될 여지가 없는가.
			2. 시각적인 이해는 좀 자존심상하는 이해임. 본질적 이해같지 않아서.
		
		수학 논리적인 이해 :
			
			이것을 이해하려면 "방향정보"가 왜 생겨나는지를 깊이 이해해야 할 것 같다.
			아마도..
			B =
				v1, v2, v3 이고
				v3 = v1 + v2 라면,
			
			A로 변환된 (특정 수를 곱하고 다시 더한) v3는 여전히 변환된 v1과 v2에 지나지 않는다..
			라는 논리가 성립하기 때문일 것 이다.
			
			**근데그구체적인메커니즘은아직도모르겠음.

```
