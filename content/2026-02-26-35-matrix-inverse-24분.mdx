```yaml
수학 :

	* 어떤 방법으로 연산을 하는지 메커니즘을 파악하는 것이 수학의 다양한 분야를 배울 때 중요하다.

Matrix :
	
	대응하는 차원의 데이터(벡터)에 대하여
	특정한 연산을 가하여 새로운차원으로 변환된 데이터를 반환하도록 설계된 연산 기법.
	
		"특정한 연산" :
			
			주어진 벡터의 대응 하는 차원의 수 만큼 matrix에 주어진 차원의 데이터로 새롭게 반환함.
			
		"Matrix Multiplication으로 가능한 데이터 연산" :
			
			변환 - 수의 갯수를 바꿈
			차원 치환 - 주어진 차원을 다른차원의 데이터로 바꿈

I :
	
		주어진 벡터에 대하여, matrix연산의 규칙을 지키면서 아무런 연산을 가하지 않음.
			* 해당벡터의 대응하는 차원의 데이터에 아무런 추가적인 연산을 가하지않고 그대로 반환함.
	

Matrix Inverse :
	
	데이터에 Matrix 연산을 가하기 이전 상태로 만듦.

	예제 : 해당 벡터에 연산을 가하고, 다시 연산이전의 상태로 만들어보세요.

	1. 연산하기

		[a b] [x]
		[c d] [y]

		->

		[ax + by] = x new
		[cx + dy] = y new

			*설명 :
				x를 변환 matrix에 넣으면 변환 규칙에 따라 1차원 정보인 a가 x만큼, 2차원 정보인 c가 x만큼 output됩니다
				y를 변환 matrix에 넣으면 변환 규칙에 따라 1차원 정보인 b가 y만큼, 2차원 정보인 y가 y만큼 output됩니다
				이제 각 차원의 정보를 더하면 x_new, y_new를 얻을 수 있습니다.

	2. 연산하기 이전상태

		[x]
		[y]

	3. 되돌리기

		* 되돌리기의 의미가 무엇인가?
			* x new와 y new에 특정한 연산을 가하여서 x new는 x로, y new는 y로 만들면 된다.

		x new = [ax + by]
		y new = [cx + dy]

		3-1) 효과를 되돌리기 위하여 inverse matrix 꼭 써야하나?

		Inverse Matrix쓰지 않고도 abcd와 x new/y new를 찾을 수 있다.
		Gaussian Elimination 활용하면 된다.

			아래의 이 결과는 linear system이기 때문이다.

				x new = [ax + by]
				y new = [cx + dy]

				ax + by - xnew = 0
				cx + dy - ynew = 0

			x의 영향력을 소거한 뒤, y가 미치는 영향력을 b와d의 비교를 통해 추론하고,
			이를 기반으로 다시 x를 찾으면,
			x,y를 추론하는 것은 어렵지 않다.

			**
			이걸 컴퓨터에게 시킬 수 있는 Logic은 필요한데, 그게 Inverse Matrix인가 싶다.
			반대로 생각하면 gaussian elimination쓰지 않고도 inverse matrix로 Linear system equation 풀어버릴수도 있겠는데. (이것이 이해의 힘인가)

	
	
	4. Gaussian Elimination과 Inverse Matrix 구하는 공식은 본질적으로 같다
	
		한 땀, 한 땀 대입법을 통해서 x_new와 y_new에 가해진 변환 연산을 역으로 추적하다보면,
		분명히 ad - cd에 대한 논리를 도출해낼 수 있을 것 이다.
		아직 해보진 않았지만.
	
	
```

```yaml
왜 역행렬과 곱하면 단위행렬이 되어야 할까.

수학적으로 설명 :

'역수' 라는 것이 있는데,
어떤 수에 그 '역수'를 곱하면 1이 나온다고 한다.

2 × (1/2) = 1

4x2 = 8인데, 1/2을 곱하면 다시 원래 수로 되돌아온다.

여기서 알 수 있는 것은,

1이라는 숫자를 숫자로 보지말고 "기준"이라고 보면,
이것을 2번 늘리는 연산을 한 뒤, 다시 "역수"로 연산하면 1이라는 "기준"으로 되돌아온다는
효과를 볼 수 있다는 것이다.

어떠한 효과를 완전하게 상쇄시켜버리는 것 이다.

	그래서 내가 이해한 중요한 성질은 :
		"역수"는 특정 수의 연산 효과를 "되돌린다는 것"이다.

행렬에서도 마찬가지일 것이다:

변환을 가했는데, 다시 이 변환의 효과를 상쇄 시켜버리고.
이 효과를 측정하기 위한 기준점이 identity matrix가 되는 것.

	그럼 identity matrix의 의미를 정확히 이해해야 할필요가 있다 :
	
	I의 의미는 "각 정보열에 할당된 차원 정보를 단 한번만 세는 것"이다.

	 변환A 1차 2차 3차 용 변환       I
			  [1  2  3]              [1  0  0] 1차원 (x)
			  [4  5  6]              [0  1  0] 2차원 (y)
			  [7  8  9]              [0  0  1] 3차원 (z)
		
		-> 1차원에 대한 벡터 정보만 가져오면 1x,5y,7z
		-> 2차원에 대한 벡터 정보만 가져오면 2x,5y,8z
		-> 3차원에 대한 벡터 정보만 가져오면 3x,8y,9z
	
		이 정보를 다 가져와서 더하면 'A' 그 자체를 다 더한것과 같다.
	
		그 외, 각 정보 열 1,2,3에게 할당된 1,2,3차원의 정보 외에 어떠한 다른 정보도 "count하지 않음"
		
		예를 들어, I의 1열은, 변환 A에서 1차원에 할당된 정보를 가져온다. 만약 I의 2차원, 3차원 정보도 존재했다면
		추가적으로 A에서 2차원, 3차원에 해당하는 정보를 가져왔겠으나, 0이므로 1차원 정보만 가져온다.
	
		결론은, I는 matrix의 정보 그 자체만 가져오도록 정교하게 설계되어 있는 표이다.
	
	
		어떻게 inverse matrix와 연관이 있는가 ? :
```

```yaml

느낀점 :
	
	1. 
	수학에 어떤 대단한 의미가 있을까 나는 탐구하려고 노력했다.
	어떤 기하학적 의미가 있을까, 도대체 I는 무슨 의미일까 등.
	더하기란 무엇인가, 나누기란 무엇인가, 이것은 어떤 행위를 하는 것인가 등.
	탐구하면 할수록 알게되었다.
	수학의 규칙 그 자체가 엄밀한 의미라는 것을.
	1 + 1 = 2 에서, '+'라는 것이 무슨 의미인가를 탐구했지만,
	그냥 +는 있는 그대로 1과 1을 2로 만들어주는 "행위"그 자체인 것 이다.
	추가적인 하나의 의미를 탐구하지 않고 그것을 있는 그대로의 규칙을 받아들일 때,
	오히려 더 큰 논리적 자유를 얻을 수 있으며, 수학의 아름다움(즉, 논리적 무결성)을 탐구할 수 있다.

	기하학적으로 의미를 정의해버리면, 오히려 한계가 뚜렷해진다.
	물론 물리적 현실에서는, 더하기라는 것은 결국
	2개를 인식한다와 같은 '인간의' 현실적 의미를 부여해볼 수 있을 것.

	그래도, 수학의 의미를 안다는 것은 나에게 도움이 되는 것 같다.
	더하기를 더하기로만 사용하지 않을 수 있게 되었으니까.
	그리고 탐구를 통해 I도, 어떤 연산을 가하는 기호인지 알 수 있게 되었으니까.
	그냥 I가 나온다, 이렇게 하는 것보다는 더 해상도가 높게 느껴진다.

	수학 연산을 의미적으로 알고 있다면, 수학 연산을 현실의 이상하게 적용하는 것은 막을 수 있지 않을까?
	원래 원리를 모르면 막 아무 상황에다가 갖다 붙이잖아 사람은.
	수학 뿐만 아니라, 여러가지 판단에서도.
	그건 그렇다.. 사람에게 1,0,1,0을 때려넣을수도 없고.
	해상도 높은 정보를 줘야하는데, 그게 의미적으로 현상을 이해하는 인간이라면,
	수학 연산에 대한 의미를 구체적으로 알고있는 행위도 바람직한 듯 싶다.

		예를 들어 :
		만약 이 개념을 배운 다른사람들에게, 변환된 벡터값 준 다음에, input vector 구해보라고하면, inverse matrix쓰지않고 gaussian elimination쓸 수 있을까?
		훨 빠른것 같음.

	생각해보니, 그렇다면 수학이거 의미적 결합만 잘 되면 되는 것 이라서.
	되게 쉽게 AI가 되지 않을까?
	다만, 수의 계산으로만은 안되고, 하나의 수학적으로 무결한 의미를 창출해서, 그 의미를 손실하지 않은 상태로
	다른 의미와 결합할 수 있어야 함.
	그래서 그냥 수학 계산만 해가지고는 AI안되고, 이 의미 결합이 매우 중요한 듯.

	2.
```

구하는 법

```yaml
예시로 이 행렬의 역행렬을 구해볼게요:

[2  1]
[1  1]

1단계: 원래 행렬 옆에 단위행렬을 붙입니다
[2  1 | 1  0]    <- 첫 번째 행
[1  1 | 0  1]    <- 두 번째 행

2단계: 두 번째 행에서 첫 번째 행의 1/2을 빼서 첫 번째 열을 정리합니다
[2  1 | 1  0]
[0  0.5 | -0.5  1]   <- (두 번째 행) - (첫 번째 행)/2

3단계: 첫 번째 행을 2로 나눕니다
[1  0.5 | 0.5  0]    <- (첫 번째 행)/2
[0  0.5 | -0.5  1]

4단계: 첫 번째 행에서 두 번째 행을 빼서 두 번째 열을 정리합니다
[1  0 | 0.5  -0.5]   <- (첫 번째 행) - (두 번째 행)
[0  1 | -1  2]       <- 완성!

이제 오른쪽 부분이 역행렬입니다:
[0.5  -0.5]
[-1    2]
```

중요한 성질

```yaml
중요한 성질:

- 모든 행렬이 역행렬을 가지는 것은 아님
- 정방행렬(n×n)만 역행렬을 가질 수 있음
- 행렬식(determinant)이 0이 아닐 때만 역행렬 존재
```
