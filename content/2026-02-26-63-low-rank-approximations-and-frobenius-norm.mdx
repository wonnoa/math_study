# 63 Low-Rank Approximations and Frobenius Norm

```yaml
Low Rank Approximation :

	무엇 :
		
		"거대한 matrix를 더 작은 matrix의 곱으로 축소."
		
		A = W * H
	
	어떻게 :
		
		"영향력이 큰 Rank (차원관계)빼고는 그냥 관계 데이터를 다 소거한다고 함.",
		"10만개의 데이터가 10만개의 관계를 표현하는 대신, k차원의 관계만 표현한다고 생각하면된다"
				
		원래 Matrix A의 크기가 10만 x 10만 이었다면,
		W (10만 * k) * H (k * 10만)으로 표현함.
			(여기서 k는 10만보다 훨씬 작은 값)
	
		핵심 :
			
			"10만보다 훨씬 작은 수인 k로 인하여 low rank라고 표현"
	
	왜 :
	
		메모리 절감
		연산속도 향상		

```

```yaml
오차 측정 :

	"실제 matrix의 각 원소에서 변조된 matrix의 각 원소값을 뺀다, 그리고 그 차이를 제곱한다",
	"제곱한 차이들을 다 더한다, 그리고 다시 더한값들을 square root한다",
	"이것이 오차다"

	원래 행렬: 
		"A (N×M 크기)"
	
	근사 행렬: 
		"A_hat = W × H"
	 
		이때 A ≈ A_hat이 얼마나 비슷한지를 **오차(Error)**로 나타내야 한다.
	
	
	Squared Error :
		"A[i,j]와 A_hat[i,j]의 차이를 제곱한 모든 합을 구한다."
		
		이것이 데이터셋의 총 오차 제곱이다.
		
	Frobenius Norm(프로베니우스 놈) :
		
		"제곱합에 Sqaure Root까지 한다"
		
			1. 데이터간 차이는 왜 구하나
				-> 얼마나 데이터간에 오차가 있는지 파악한다
			
			2. 제곱은 왜 하나
				-> 절대값으로 만들 수 있다.
				-> 절대거리 구할 수 있다.
					(x제곱 + y제곱을 sqrt하면 origin으로부터 직선거리 나온다.)
					(그 차이 보겠다는 거다)
			
			3. sqrt왜 하나
				-> 데이터 포인트들의 차이의 직선거리 보겠다
			
```

```yaml
벡터간 유클리드 거리 :

sqrt ((x1-x2)^2 + (y2-y1)^2)

	의미. 벡터간의 차이로 발생하여 만들어진 x축과 y축의 직선거리를 직각삼각형의 변으로 하여,
	대각선의 거리가 벡터간의 거리.
```
