# 23 Matrix Multiplication (18분)

- matrix multiplication의 정의
    
    ```yaml
    정의 :
    "첫 번째 행렬의 행(row)과 두 번째 행렬의 열(column)을 대응되는 성분끼리 곱한 후, 그 결과를 모두 더해서 새로운 행렬의 원소를 만드는 연산",
    
    의미를 부여한 이해 :
    
    "벡터를 주어진 규칙에 따라 새로운 원소들로 output한 뒤 모든 숫자를 더함"
    	
    	기존 벡터 x,y라는 수는 기존의 (1,0),(0,1)의 unit vector를 x,y만큼 "count"하는 것 이었는데,
    	unit vector가 바뀜에 따라, x,y도 새로운 unit을 count하게 변한다.
    
    	상대성이론과 같은 원리처럼 기존 물체는 놔두고, 
    	물리적 공간이 바뀐다는 연결점을 생각해 볼 수도 있다..
    	숫자는 그대로인데, 단위가 바뀌었음.
    
    예 :
    
    Vector v (x1,y1) = [3 2]
    
    A [2 1]
    	[1 3]
    
    A의 세로가 new x,y 값을 나타낸다.
    
    그렇다면, new vector는
    
    [3 x 2 + 2 x 1] (new x)
    [2 x 1 + 2 x 3] (new y)
    
    =
    
    [8,9] 이다
    
    이것은,
    
    "**벡터 v와 matrix 각 row의 dot product와 같다.",
    "하지만 수학적 규칙은 행과 열의 숫자의 곱연산 후 덧셈이다",
    "같은 원소끼리만 연산하지는 않는다"**
    
    ```

    		어떤 "의미"를 부여할 수 있겠는가? 원소끼리 곱하고 더하는건 알겠는데 말이다.**

- how to conduct matrix multiplication
    
    ```yaml
     : "matrix multiplication is a generalization of the dot product"
    
    원래 그냥 행과 열을 각각 dot product하는 것이 원칙이긴함.
    그런데 연산해보면 dot product보다 transformation 관점이 더 편하다 나한테는..
    
    1. 두 행렬을 곱하면 새로운 행렬이 나오며 이를 C라고 부를 수 있다 :
    
    AB 일경우, B에 A가 적용되어 C가된다 라고 생각할 수 있다.
    
       차원 1  차원 2     정보 1  정보 2
    A = [1     2]    B = [5      6]     C = [c11 c12]
        [3     4]        [7      8]         [c21 c22]
    
    	
    2. matrix multiplication은 C의 각 "원소"들을 구하는 과정이다 :
    	
    	A의 row와 B의 column의 corresponding하는 원소들을 곱하고 더하기 연산해야한다.
    	
    	C11 (1행 1열)을 구하려면: A의 1행 [1 2]와 B의 1열 [5 7]을 사용
    	C12 (1행 2열)을 구하려면: A의 1행 [1 2]와 B의 2열 [6 8]을 사용
    
    3. 각 위치 값은 아래와 같은 방식으로 계산한다 : 
    	
    	"내적이라고 불러도 되고, 변환으로 불러도 된다.",
    	"중요한 것은 2개의 차원 데이터로 하나의 변수 값을 새로이 구하는 과정이라는 것."
    	
    	C11 = 1×5 + 2×7 = 5 + 14 = 19
    	C12 = 1×6 + 2×8 = 6 + 16 = 22
    	C21 = 3×5 + 4×7 = 15 + 28 = 43 
    	C22 = 3×6 + 4×8 = 18 + 32 = 50
    	
    	따라서,
    	C = [19 22]
    	    [43 50]
    	
    	** 규칙에 따라 벡터간 dot product 연산을 한 것임.
    		이걸 정보가 변환되는 관점으로 볼 것이냐, 그냥 dot product하는 것으로 볼 것이냐는
    		어떻게 정보를 인식할 것인가의 차이일 뿐 
    	
    	** 왜 각 값이 이렇게 위치되는가 ? :
    	
    	B matrix의 '정보 1'의 x와 y값이 각각 변환 matrix A와 연산되어 새로운 정보를 산출한다.
      변환 규칙에 따라 각각 차원에 알맞는 변환로직과 연산되어 새로운 정보를 산출해낸다.
    	예를 들어, B의 정보 1 [5,7]은 각각 1차원과 2차원에 대한 정보를 담고 있는 정보이다.
    	5라는 1차원 정보는 A에 input되어 새로운 1차원, 2차원 정보인 '5곱하기 1'과 '5곱하기 3'을 각각 산출한다.
    	7이라는 2차원 정보는 A에 input되어 새로운 1차원, 2차원 정보인 '7곱하기 2와' '7곱하기 4'를 각각 산출한다.
    	
    	이제 우리는 새로운 1차원, 2차원 정보를 얻었으므로, 기존의 정보 1,2가 새롭게 변환된 
    	new 정보 1,2 자리에 각각 1차원 2차원 값을 다시 기록해야한다.
    	
    	   차원 1  차원 2      정보 1  정보 2         new 정보 1  new 정보 2
    		A = [1     2]    B = [5      6]      C = [c11        c12]
    		    [3     4]        [7      8]          [c21        c22]
    
    	따라서, 
    	
    	정보 1에서 산출된 1차원 정보인 '5곱하기 1' 과 '7곱하기 2' 를 각각 더하고 (선형 벡터이므로)
    	정보 1에서 산출된 2차원 정보인 '5곱하기 3' 과 '7곱하기 4' 를 각각 더하여
    	각각 차원에 맞는 위치정보에 기록한다.
    	
    	이것이 matrix multiplication을 진행하는 법칙을 언어로 기술한 것이다.
    	
    	
    	나에 대한 생각 :
    	
    	.. 난 이렇게 언어로 의미를 기술하지 않고 단순히 "수"의 규칙만 나열하면 그것을 이해했다고
    	말할 수 없다.. 
    	규칙을 그대로 받아들이는 이과와, 그것에서 의미를 언어로 탐색하는 문과의 차이인가?
    	문이과 차이는, 결국 어떤 언어도구로 세상의 규칙을 탐구하느냐인 것 같다.
    	
    	'수'는 새롭게 의미가 부여되어져야하고, '언어'는 이미 의미가 부여되어져있다.
    	이것이 철학의 쓸모가 아닐까 한다.
    	수만 있으면 인간에게 공허하고, 언어만 있으면 규칙이 정교하지 못하다.
    	
    	그래서 아래 규칙으로 설명하면 이제 나는 "혼란스러워지기 시작한다"
    	그대로 규칙을 받아들이는 것은 어렵지 않지만, 그것이 무슨 의미인지 알 수 없다.
    	
    4. C 행렬의 i row j column에 들어갈 값을 구하는 공식 :
    
      Cij = Σ(aik × bkj) :
      
    	Σ k=1부터 k까지 : "이 과정을 k번 반복해서 더하라"는 의미
    	aik: A행렬의 i행에 있는 모든 값들
    	bkj: B행렬의 j열에 있는 모든 값들
    	
    	각각 row와 column의 모든 값들을 곱하고 더하는 것.
    	
    	해석 :
    	
    	corresponding하는 원소들의 dot product이다.
    	
    	Cij는 단순히 행과 열을 넘어, 어떤 정보를 기술하고 있는지에 대한 정보를 담는다.
    	
    	i = 차원의 정보
    	j = input한 정보 no.
    	
    	A       ×     B      =     C
    	[1 2]   ×   [5 6]    =   [c11 c12]
    	[3 4]       [7 8]        [c21 c22]
    	
    	이라면, Cij는
    	A의 i행과 B의 j열을 내적하면 된다.
    	내적이 뭔데? - corresponding 하는 원소들끼리 곱해서 더하는 것이다.
    	왜 corresponding할까?
    	
    	예를 들어,
    	c12에는 1차원 정보, B의 정보 2번 이라는 의미를 담고 있다.
    	
    	그래서, 이것을 구하려면.
    	6이랑 1곱해서 1차원 정보값 구하고.
    	8이랑 2곱해서 또 2차원 정보값 구하면 된다.
    	그리고 더한다.
    	지금 이 과정이 기호로 표현된 Cij = Σ(aik × bkj) 이다.
    	
    	예 :	  Cij = Σ(aik × bkj)
    	      c22 = Σ(a2k × bk2), k는 1부터 2까지
    	
    	Σ  = 모두 더하라
    	
    	aik = matrix a의 i행의 k번째 숫자
    	bkj = matrix b의 j열의 k번째 숫자
    	
    	k = 행이나 열에 존재하는 index의 갯수 (1부터 n까지)
    	
    	= dot product : 
    		
    		즉 해당 차원과 정보에 알맞는 정보들의 더한 값.
    	
    
    	여튼 그래서 c12에는
    	정보 2의 변환된 1차원값들의 합이 들어있다.
    	이것은 해석에 따라 dot product에서 산출된 scalar이기도 하고,
    	새로운 벡터 1차원 좌표이기도 하다.

        이렇게 표현해야하는게 규칙임?

        8초 동안 Reasoned about matrix representation 규칙
        
        표현 방식은 상황에 따라 달라질 수 있습니다.
        
        - **수학적 관점:**문제에서 A와 B의 형태를 보면, A는 3개의 2차원 column vector로, B는 3개의 2차원 row vector로 표현되어 있습니다. 이 표현은 두 행렬의 내적을 계산하기 위해 내적에 필요한 두 벡터의 방향(열 vs. 행)을 맞춘 것입니다.
            
            AA
            
            BB
            
            AA
            
            BB
            
        - **데이터 분석:**데이터 분석에서는 보통 관측치(샘플)를 행으로, 변수(피처)를 열로 배치하는 경우가 많습니다. 이 경우, 특정 연산(예: 공분산 행렬 계산, 선형 회귀 등)을 위해 데이터를 transpose하는 경우가 있지만, 반드시 항상 transpose해야 한다는 규칙은 없습니다.
            - 상황에 따라 데이터의 형태를 맞춰주기 위해 transpose를 사용합니다. 예를 들어, X가 n×p (n개의 샘플, p개의 피처)일 때, 피처 간의 상관관계를 계산하기 위해 X⊤를 사용하는 경우가 있습니다.
                
                XX
                
                n×pn \times p
                
                X⊤X^\top
                
            - 따라서 **목적과 연산에 따라 데이터의 배열(행 vs. 열)을 조정할 필요가 있을 때 transpose를 사용**할 뿐, 이것이 항상 필요한 것은 아닙니다.
        
        즉, 데이터 분석 시에도 내적이나 행렬 연산을 올바르게 수행하기 위해 데이터의 배치를 확인하고 필요에 따라 transpose를 적용하면 됩니다.

        **행렬 곱셈의 경우에는**

        - (AB)_{ij} = \text{(왼쪽 행렬의 \(i번째 행 벡터)} \cdot \text{(오른쪽 행렬의 j번째 열 벡터)} )
            
            jj

            [1 3]
    
    A(B + C) = [1×3 + 2×1  1×1 + 2×3]
               [2×3 + 1×1  2×1 + 1×3]
    
             = [5 7]
               [7 5]
    
    AB + AC 계산 :
    
    AB = [1×2 + 2×1  1×1 + 2×2]
         [2×2 + 1×1  2×1 + 1×2]
         
       = [4 5]
         [5 4]
    
    AC = [1×1 + 2×0  1×0 + 2×1]
         [2×1 + 1×0  2×0 + 1×1]
         
       = [1 2]
         [2 1]
    
    AB + AC = [5 7]
              [7 5]
    ```

- inner dimensions must match 에 대한 의문과 해결
    
    의문
    
    ```yaml
    의문 :
    "inner dimension만 match 하면, 벡터 x의 y,z,... 등의 값은 그냥 소실되는 것인가?"
    
    의문에 대한 설명 :
    
    	Vector X : 
    		[4 7]
    	
    	Transformation A :
    		[4 7]
    		[5 8]
    	
    	일 때
    	
    	일반적으로 우리는 변환을 이렇게 한다 :
    		
    		"새로운 단위를 이용하여 기존벡터를 다시 센다"
    		
    		기존 벡터의 현재 x값과, 새로운 x의 좌표를 곱한다
    		기존 벡터의 현재 y값과, 새로운 y의 좌표를 곱한다
    		마지막으로 새롭게 얻어지는 x 좌표들과 y좌표들끼리 더한다
    		
    	계산 :
    		
    		[Xx = 16 Yx = 49]
    		[Xy = 20 Yy = 56]
    	
    	
    	"inner dimensions must match를 의미적으로 풀어보면" :
    	
    	변환이라는 과정은, 기존 좌표의 값들의 단위값이 달라져서, 그것들의 수를 다시 세어서 변한값들을 더해야하는 상태이다.
    	
    	내가 벡터의 x와 y좌표(값)를 가지고 있을 때, 각 x와 y를 변환시킬 새로운 '값'이 필요할테니,
    	가로 x 세로 수가 같은 것이 아니라,
    	가로 x 가로 수가 같아야 하는 것 아닌가?
    	
    	그래야 새로운 x와 y를 "다시 셀 수 있는 것" 아닌가?
    	
    	실험 :
    	벡터 X - [4 7]
    	
    	Case 1. 가로로만 변환 값을 가진 경우 A가 [1,1]
    	Case 2. 세로로만 변환 값을 가진 경우 A가  [1]
    																		 [1]
    
    																
    		"Case 1" : A가 [1,1]일 때
    		
    		좌표 X와 Y에 대한 새로운 X값만 가지고 있는 상태이다.
    		New Vector는 [11, ?]으로 y좌표가 없다.
    		이것은 차원 축소되는 것 인가? 0도 아니고 그냥 소멸되어버렸다.
    		
    		
    		"Case 2" : A가 [1]일 때
    									 [1]
    		
    		기존 Vector X = [4 7]은 y값을 잃어버리게 된다.
    		기존 x좌표에 대한 변환값은 가지고 있지만, y좌표에 대한 변환값은 없다.
    		 														
    	
    	AI 말로는 2번 케이스의 경우 그냥 세로 [1] 에, 벡터 X의 각각의 x,y값을 곱하라고 한다.
    																[1]
    	
    	(1x4) + (1x7) 하란다.
    	
    	벡터 y의 값을 왜 x가 곱해져야 하는 component와 곱하는지 이해할 수 없다.
    ```
    
    outer dimension이 다를 때, 차원 축소
    
    ```yaml
    : "V의 각 [x,y,z]축의 값들이 x,y축의 값만 가지게 된다."
    ```
